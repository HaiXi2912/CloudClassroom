# 01 环境与镜像源（Windows）

> 本项目为前后端分离：后端 SpringBoot 2.7 + MyBatis Plus + MySQL 8；前端 Vue 3 + Vite + Element Plus。
> 要求：全部中文注释/文档/页面内容，且尽量使用中国境内镜像源，保证依赖可下载。

## 1. 推荐版本（已按本项目固定）

- JDK：17（SpringBoot 2.7 推荐且兼容）
- Maven：3.9+
- Node.js：18+（当前你是 22.x 也可用）
- MySQL：8.0+（本项目默认 `localhost:3306`，密码 `123456`）

## 2. 当前工作区本地工具路径（本项目自带）

为了避免你电脑全局环境不一致，本项目把 JDK/Maven 安装在工作区：

- JDK17：`d:\CloudClassroom\.devtools\jdk\jdk-17.0.16\bin\java.exe`
- Maven：`d:\CloudClassroom\.devtools\maven\maven-3.9.12\bin\mvn.cmd`

后续运行后端时，优先使用上面的路径，而不是系统自带 `java`/`mvn`。

## 3. Maven 国内镜像配置

本项目提供了一个 Maven 配置文件模板：

- 文件：`tools/maven/settings.xml`
- 用途：把依赖仓库镜像到阿里云（国内访问更稳定）

### 3.1 使用方式

在 PowerShell 中运行（后端构建时会用到）：

```powershell
cd /d d:\CloudClassroom
$env:MAVEN_OPTS='-Dfile.encoding=UTF-8'
.\.devtools\maven\maven-3.9.12\bin\mvn.cmd -s tools\maven\settings.xml -v
```

## 4. NPM / PNPM 国内源

本项目提供：

- `.npmrc`：把 npm registry 指向 `https://registry.npmmirror.com`

你也可以全局设置（可选）：

```powershell
npm config set registry https://registry.npmmirror.com
pnpm config set registry https://registry.npmmirror.com
```

## 5. MySQL 连接约定（本项目默认）

- 地址：`jdbc:mysql://localhost:3306/cloud_classroom?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai&useSSL=false&allowPublicKeyRetrieval=true`
- 用户名：`root`
- 密码：`123456`

> 注意：你本机 MySQL 是否是 root/123456 需你确认；如果不是，后续我会提供一键修改配置的方法。

## 6. 验证点（必须跑通才进入下一步）

- `java -version` 显示为 17
- Maven 能输出版本：`mvn -v`
- 前端能正常 `pnpm install`
- 后端能正常下载依赖（后续第2步后验证）
